# ルール
- コメントは日本語で書くこと
- 実装完了後は必ず以下を確認してから完了報告すること
  1. `npm run build` でビルドエラーがないことを確認
  2. 開発サーバーを起動して Preview MCP で動作確認
  3. 主要な機能 (構築検索・ポケモン絞り込み・形式切り替え) が正常に動くことを確認
  4. エラーが出た場合は自分で修正してから完了報告する
- 作業開始前の git コミットは不要。指示された作業だけを実行すること

# プロジェクト概要
ポケモンチャンピオンズの構築紹介一覧サイト。
バトルデータベース風の検索 UI で、シングル/ダブル切り替えとポケモン名絞り込みが可能。
MVP ではダミーデータで骨組みを固め、後から Supabase や外部入力に差し替える設計。

# スタック
- Next.js 15 (App Router) + TypeScript + Tailwind CSS
- MVP は DB なし (ダミー JSON)
- ポケモン名マスターと日本語ファジーマッチは `../pokemon-damage-calc/lib/` から流用
